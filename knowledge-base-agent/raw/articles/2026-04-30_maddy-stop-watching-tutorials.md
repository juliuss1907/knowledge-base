# Stop Watching Tutorials. Do This Instead

## Tóm tắt

Maddy Zhang giải thích vì sao hầu hết developers học technology mới theo cách sai — và đề xuất phương pháp thay thế hiệu quả hơn.

---

## Các chapter chính

### 1. Why Developers Get Stuck [0:00]
Vấn đề gốc: Developers cứ watch tutorials liên tục nhưng không tiến bộ.

### 2. Tutorial Hell [1:02]
Bẫy "Tutorial Hell" — lúc nào cũng xem tutorial, copy code theo mà không hiểu sâu. Kết quả: khi gặp problem thực tế thì stuck.

### 3. Start Building Early [3:20]
Bắt đầu build project càng sớm càng tốt — không cần hiểu hết 100% tutorial mới được code.

### 4. 90-Minute Deep Work [4:20]
Session deep work 90 phút không bị gián đoạn — tránh context switching làm giảm hiệu suất học.

### 5. Interleaving Learning [5:20]
Học xen kẽ nhiều topics thay vì học 1 thứ từ đầu đến cuối rồi mới sang cái mới. Ví dụ: 30 phút React, 30 phút Python, 30 phút System Design — xen kẽ giúp não巩固 kiến thức tốt hơn.

### 6. Using AI to Learn Faster [6:50]
Dùng AI (ChatGPT, Claude) để:
- Giải thích concepts đang học
- Debug code
- Tạo exercises thực hành
- Review code

### 7. Document What You Learn [8:10]
Viết documentation cho những gì đã học — giúp củng cố kiến thức và tạo reference cho bản thân sau này.

## Quote nổi bật

> "If you're constantly trying to learn the new thing the moment it drops, then you're not being productive. Your career is based on how productive you are, not how big a nerd you are."
