# Source: Spec-Driven Development with AI (GitHub Spec Kit)

## Tóm tắt
GitHub ra mắt Spec Kit để chuyển đổi từ "vibe coding" sang "spec-driven development". Quy trình gồm 4 phase: Specify $\rightarrow$ Plan $\rightarrow$ Tasks $\rightarrow$ Implement.

## Key Points
- **Vấn đề:** Vibe coding không reliable cho production vì agent phải guess requirements.
- **Spec Kit Workflow:** 
  - `/specify`: Tạo detailed spec từ high-level prompt.
  - `/plan`: Tạo technical plan dựa trên stack/constraints.
  - `/tasks`: Chia nhỏ thành các chunk nhỏ, testable.
  - `/implement`: Triển khai từng task.
- **Triết lý:** "Intent is source of truth" thay vì "Code is source of truth".
