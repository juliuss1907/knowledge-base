# Vibe Coding and the Maker Movement
**Author:** Technically  
**Link:** https://open.substack.com/pub/technically/p/vibe-coding-and-the-maker-movement  
**Date:** February 26, 2026
---
## Tóm tắt
Bài viết so sánh **vibe coding** với **Maker Movement** (~2005-2015) và phân tích tại sao metaphor "transformation-through-making" không còn phù hợp với vibe coding.
---
## Parallels với Maker Movement
| Maker Movement | Vibe Coding |
|---|---|
| 3D printers, breadboards | Claude Code, AI agents |
| Crapjects (useless 3D prints) | Slop (AI-generated code) |
| Scenius phase — years of playground | **Skipped entirely** — straight to production |
| Internet-native network intellectuals | Similar figures emerging |
---
## The Scenius Problem
- **Maker Movement:** Có "scenius phase" — period where small groups play with tools trước khi có economic output expectations
- **Vibe coding:** Skip this phase, deploy directly to enterprise codebases
- **Consequence:** No protected playground, no time to accumulate weird/playful knowledge
- Result: **Hypomania** — productive capacity tăng nhưng evaluative faculty chưa developed
> "You lose the ability to distinguish between 'this is good' and 'I feel good making this.'"
---
## Value Accumulation Pattern
**Maker Movement end:**
- 3D printers commodified **prototyping**
- But manufacturing knowledge accumulated in Shenzhen
- Cheap tools democratize one layer → layer beneath captures surplus
**Vibe coding now:**
- People prototype tools threatening SaaS business models
- Value flows upward to model layer, training data, infrastructure
- Vibe coders risk becoming interchangeable
---
## New Metaphor: Consumption
Vibe coding = **consumption of surplus intelligence**
AI là cognitive energy available whether you use it or not. Vibe coding channels this surplus into play, exploration, rapid creation.
**4 productive ways to consume:**
1. **Taste/Judgment** — Pattern recognition từ việc build & discard nhiều prototypes → creative direction, curation
2. **Spectacle/Attention** — Performance of making generates audience, reputation
3. **Gift Economy** — Open source tools create social bonds, network position
4. **Informational Exhaust** — Document failures, edge cases → data fortress
---
## Key Insight
> "The consumption framing sidesteps burnout entirely. You're not reaching inward. You're starting from the position that there is extra energy available and it needs to go somewhere. The question shifts from 'what does this say about me as a maker' to 'what's the most interesting thing I can spend this on.'"
