# Source: Spec-Driven Development with Claude Code

## Tóm tắt
Kinh nghiệm thực tế sử dụng spec-driven development với Claude Code để giảm tech debt và tăng hiệu quả lập trình AI.

## Key Points
- **3 mức độ:** Spec-first (viết trước), Spec-anchored (giữ để evolve), Spec-as-source (chỉ edit spec).
- **Kinh nghiệm Claude Code:** 
  - Sử dụng Sonnet 4.6 để tối ưu quota.
  - Chia nhỏ build, test sớm qua stacked PRs.
  - Luôn update spec khi thay đổi design.
- **Kết luận:** Đầu tư vào planning giúp giảm thiểu việc sửa lỗi (course correct) sau này.
