# Vibe Coding và Maker Movement

## Mô tả

So sánh **vibe coding** với **Maker Movement** (~2005-2015). Điểm khác biệt quan trọng: Maker Movement có "scenius phase" — period để play với tools trước khi có economic output expectations. Vibe coding skip hoàn toàn phase này, deploy thẳng vào production.

## Parallels

| Maker Movement | Vibe Coding |
|---|---|
| 3D printers, breadboards | Claude Code, AI agents |
| Crapjects | Slop (AI-generated code) |
| Scenius phase | **Skipped** — straight to production |

## The Scenius Problem

Result: **Hypomania** — productive capacity tăng nhưng evaluative faculty chưa developed. "You lose the ability to distinguish between 'this is good' and 'I feel good making this.'"

## New Metaphor: Consumption

Vibe coding = **consumption of surplus intelligence**

4 productive ways to consume:
1. **Taste/Judgment** — Pattern recognition từ build & discard nhiều prototypes
2. **Spectacle/Attention** — Performance of making generates audience
3. **Gift Economy** — Open source tools tạo social bonds
4. **Informational Exhaust** — Document failures → data fortress

> "You're starting from the position that there is extra energy available and it needs to go somewhere."
